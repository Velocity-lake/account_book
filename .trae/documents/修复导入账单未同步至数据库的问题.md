# 修复导入账单后列表不显示的问题

## 问题分析
用户反馈“导入账单”成功但“账单列表”不显示。经排查，系统默认使用 SQLite 作为存储后端，但导入逻辑（`ui_main.py` 和 `ui_bill_list.py` 中的相关方法）仅更新了 JSON 文件 (`ledger.json`)，**未同步更新 SQLite 数据库**。而账单列表页面 (`ui_bill_list.py`) 优先从 SQLite 读取数据，导致新导入的数据不可见。

## 解决方案
在 `storage.py` 中增加同步数据到 SQLite 的接口，并在所有涉及批量导入的地方调用该接口。

## 实施步骤

### 1. 修改 `storage.py`
增加两个辅助函数：
- `sync_batch_to_db(rows)`: 将批量交易记录插入 SQLite 数据库。
- `clear_all_transactions_db()`: 清空 SQLite 中的所有交易记录（用于“覆盖导入”功能）。

### 2. 修改 `ui_main.py`
在 `ImportPanel._handle_import_rows` 方法中：
- 收集所有新成功添加的交易记录。
- 在保存 JSON 状态 (`save_state(s)`) 后，调用 `sync_batch_to_db` 将这些记录同步到 SQLite。

### 3. 修改 `ui_bill_list.py`
针对以下三个涉及导入的方法进行修复：
- `_watch_and_import_xlsx` (Excel批量录入): 增加 `sync_batch_to_db` 调用。
- `import_standard_xlsx_file` (导入标准Excel): 增加 `sync_batch_to_db` 调用。
- `import_override` (导入并覆盖): 
    - 在清空 `s["transactions"]` 时，调用 `clear_all_transactions_db` 清空数据库。
    - 在导入完成后，调用 `sync_batch_to_db` 写入新数据。

## 验证计划
- 执行修改。
- 模拟导入流程（由于无法操作 GUI，将通过代码检查确保逻辑闭环）。
- 确保所有写入 `s["transactions"]` 的地方都有对应的 DB 同步操作。
