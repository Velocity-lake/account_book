## 目标
- 当删除某个消费类别时，弹出提醒：是否同时清空历史账单中使用该类别的“消费类别”字段。
- 若选择“是”，则把所有相关账单的 `category` 置为空字符串，并从类别列表移除该类别；界面刷新后，类别按钮消失、账单列表对应格子为空。
- 若选择“否”，行为与当前一致：仅删除类别列表项，不影响历史账单。

## 改动点
- `ui_record_page.py:342-349` 的 `delete_category(self, name)` 增加提醒与可选批量清空逻辑。
- 使用现有存储接口：`storage.load_state/save_state/delete_category/backup_state`；批量清空直接遍历 `state["transactions"]` 更新 `t["category"] = ""`。

## 实现步骤
1. 读取状态并统计使用次数：`count = sum(1 for t in s["transactions"] if (t.get("category") or "") == name)`，用于提醒文案。
2. 弹框确认：`messagebox.askyesno("删除类别", f"该类别在 {count} 条账单中使用。是否同时清空这些账单的消费类别？")`。
3. 若用户选择“是”：
   - 调用 `backup_state()` 生成备份。
   - 遍历交易 `for t in s["transactions"]: if (t.get("category") or "") == name: t["category"] = ""`。
   - 调用 `delete_category(s, self.scene, name)` 从当前场景类别列表移除该项。
4. 若选择“否”：
   - 直接 `delete_category(s, self.scene, name)`。
5. `save_state(s)` 持久化；若当前选中类别正好是该项，则 `self.category = ""`；最后 `self.switch_scene(self.scene)` 刷新界面。

## 重要说明
- 不使用 `rename_category(..., new="", update_history=True)`，避免在类别列表中产生空字符串项；直接对 `transactions` 做清空更直观。
- 清空动作不区分账单类型 `ttype`，只按 `category` 匹配，符合“清除该消费类别”的直觉与需求。
- 批量操作前先 `backup_state()`，防误操作可回滚。

## 验证
- 在记录页右键任意类别选择“删除”：
  - 弹窗显示该类别被使用的条数；选择“是”后，账单列表中该类别列变为空，类别按钮消失；选择“否”后，仅按钮消失，账单不变。
- 打开账单列表（右键“进入该类别清单”前）无法再找到该类别；筛选空类别可见被清空的记录。
