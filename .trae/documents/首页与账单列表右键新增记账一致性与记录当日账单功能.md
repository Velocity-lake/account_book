## 背景与现状
- “首页预览”右击日期菜单已包含“记录当日账单”，并弹出统一的 `AddTransactionDialog`（`ui_dashboard.py:1402-1415`、`ui_dashboard.py:904-916`）。
- “账单列表”右击行目前仅有“编辑”“删除”等项（`ui_bill_list.py:190-208`），编辑时同样使用 `AddTransactionDialog`（`ui_bill_list.py:758-766`）。
- `AddTransactionDialog` 标题为“新增记账”，样式统一（`ui_add_dialog.py:9-18`），支持通过 `initial` 预填日期与类型（`ui_add_dialog.py:76-90`）。

## 修改目标
- 保持“首页预览”右击新增记账与“账单列表”右击编辑中的新增记账模块样式一致（两处统一使用 `AddTransactionDialog`，无需额外样式改动）。
- 在“账单列表”中右击某一行，增加菜单项“记录当日账单”：
  - 打开 `AddTransactionDialog`，日期预填为该行“交易时间”的日期（同日中午 12:00）。
  - 其它字段用户自选，默认“所属类别”为“支出”。

## 具体改动
- 文件 `ui_bill_list.py`
  - 在行级右击菜单中新增：`self.menu.add_command(label="记录当日账单", command=self.on_record_for_row)`（位置建议在“编辑”“删除”之后）。
  - 新增方法 `on_record_for_row()`：
    - 读取当前选中行的 `values`，取索引 0 的“交易时间”（格式 `YYYY-MM-DD HH:MM`）。
    - 构造 `initial = {"time": f"{date_str}T12:00:00", "ttype": "支出"}`。
    - 调用：`dlg = AddTransactionDialog(self, get_account_names(self.state), initial=initial)`，`self.wait_window(dlg)`。
    - 若有 `dlg.result`：
      - `new = dict(dlg.result); new["id"] = gen_id()`。
      - 同步类别字典：若 `normalize_ttype(new["ttype"])` 为收入/支出类且有 `category`，调用 `add_category(...)`（对齐 `ui_dashboard.py:1421-1425` 与 `ui_bill_list.py:771-777` 的逻辑）。
      - 调用 `apply_transaction_delta(self.state, new, 1)`，将 `new` 追加到 `self.state["transactions"]`，`save_state(self.state)`，最后 `self.reapply_last_filters()` 刷新列表。

## 验证方法
- 在“账单列表”选择任意一条记录，右击出现“记录当日账单”。
- 点击后弹窗为“新增记账”，日期已预填为该行日期（时间 12:00），“所属类别”默认选为“支出”。
- 填写其它字段并确定后：新记录出现在当前筛选结果中；账户余额与底部统计同步更新。
- 在“首页预览”右击日期使用的弹窗与上述弹窗样式一致（同一对话组件）。

## 风险与兼容
- `Treeview` 的 `values` 顺序与 `self.all_columns` 一致，本改动依赖索引 0 为“交易时间”（`ui_bill_list.py:100-103`、`116`）。若用户自定义了显示列，仅影响可见性，不影响 `values` 顺序。
- `AddTransactionDialog` 默认类型为 `TRANSACTION_TYPES[0]`（“收入”，`models.py:5-11`），因此必须通过 `initial["ttype"] = "支出"` 明确覆盖，已在方案中处理。
- 日期解析容错：若该行时间缺失或格式异常，将提示错误并中断新增（与现有 `on_ok` 的错误处理一致）。
