## 目标
- 在全局所有“账户选择”的下拉框末尾增加项：`新增账户`。
- 当用户选择该项时，弹出账户编辑对话框（复用 `AccountDialog`），完成新增后自动刷新下拉选项并选中新账户。
- 保持原有分组逻辑与只读选择体验，避免引入第三方依赖。

## 受影响位置（下拉框与行号）
- 记账页面（`ui_record_page.py`）
  - 单账户选择：`cb_account`（62–69、100–101、205–218）
  - 转账选择：`cb_from`/`cb_to`（181–195、196–204、272–288）
  - 账户分组类型选择：`cb_acc_group`/`cb_from_group`/`cb_to_group`（与账户列表联动）
- 新增/编辑账单弹窗（`ui_add_dialog.py`）
  - `cb_account`、`cb_to`、`cb_from`（35–43）
- 账单列表批量修改账户（`ui_bill_list.py`）
  - 批量对话框中的 `cb_account`、`cb_from`、`cb_to`（869–877）
- 默认账户选择对话框（`ui_main.py`）
  - `ask_default_account` 中的分组+账户（489–526），为 `cb_acc` 的 `values` 追加 `新增账户`。

## 行为与流程
- 候选列表：`values = 账户名称列表 + ["新增账户"]`
- 选择触发：绑定 `<<ComboboxSelected>>`，若值为 `新增账户`：
  1. 弹出 `AccountDialog(self, None)`（来自 `ui_account_manager.py:233–277`）。
  2. 校验并保存：`add_account` → `save_state`。
  3. 刷新当前页面的账户列表值，并将该下拉框设置为新账户名。
  4. 对于分组选择场景（如记账页/转账页），用当前分组作为类型预填到对话框（若用户修改类型则以用户为准）。

## 具体改动点
- 记账页面：
  - `_fill_all_accounts` 与 `_update_accounts_by_group` 补充在 `values` 末尾追加 `新增账户`；
  - 在 `cb_account`/`cb_from`/`cb_to` 的选择事件中增加分支：当选中 `新增账户` → 打开 `AccountDialog`，保存后刷新 `get_account_names`/`list_accounts_by_type` 并选中新增项；
  - 若当前勾选了“冻结资产”，新增账户不受影响（仅修改 `accounts` 列表）。
- 新增/编辑账单弹窗：
  - 初始化时为 `cb_account`/`cb_to`/`cb_from` 的 `values` 追加 `新增账户`；
  - 绑定选择事件，同上述流程；保存后更新本对话框的 `values` 与选中值。
- 账单列表批量修改：
  - 对话框初始化为三处账户下拉追加 `新增账户`；
  - 选择 `新增账户` 时弹窗新增并刷新三处 `values`；
  - 后续批量更新逻辑保持不变。
- 默认账户选择对话框：
  - `cb_group` 变更时填充 `cb_acc` 的 `values = by_type[group] + ["新增账户"]`；
  - 若选中 `新增账户`：以当前 `group` 作为账户类型预填打开 `AccountDialog`；保存后刷新 `by_type` 与 `cb_acc`，并选中新账户。

## 文本与一致性
- 统一使用中文项名：`新增账户`。
- 对话框标题沿用 `账户编辑`（`ui_account_manager.py:237`）。

## 错误处理与边界
- 用户取消新增：保持原选择为空或回滚到此前值；
- 名称为空或余额非数字：`AccountDialog` 原有校验会提示（`ui_account_manager.py:266–277`）；
- 新类型：自动加入 `state['account_types']`（复用现有逻辑）。
