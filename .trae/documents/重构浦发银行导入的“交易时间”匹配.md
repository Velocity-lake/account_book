## 目标

* 仅使用浦发账单 C 列（格式 `YYYY/MM/DD HH:MM:SS`）作为系统“交易时间”。

* 废弃此前所有关于浦发时间的兼容与回退逻辑（不再读取“格式化时间/交易日期/交易时间/紧凑数字/中文日期”等）。

## 代码改动

* `importers.py:map_spdb`

  * 时间：

    * 读取 `列2`（别名对应 C 列）或 `C`，若为空直接跳过该行。

    * 调用 `utils.parse_datetime` 解析，仅接受 `YYYY/MM/DD HH:MM:SS` 及其等价空格变体（函数已支持 `%Y/%m/%d %H:%M:%S`）。

    * 删除/禁用现有的时间来源回退与拼接逻辑：`格式化时间/Formatted Time`、`交易日期+交易时间`、中文日期、“紧凑数字”解析。

  * 金额：保持从 `列5/F` 读取，以符号判断“收入/支出”，金额取绝对值。

  * 备注：保持合并 `列4/E`、`列7/H`、`列9/J`，以 `" | "` 连接。

  * 账户：若存在名为“浦发银行”的账户自动填充，否则留空，走默认账户选择流程。

  * 表头行：保留跳过表头的判定（避免把“交易金额”等字符串当数据），但不再用于决定时间列索引。

* 读取层

  * 不改动 `read_csv/read_xlsx`，继续生成按表头键的字典并附加 `列{i}` 别名（我们只依赖 `列2/列5/列4/列7/列9`）。

* 其它文件不变（微信/支付宝/标准模版）。

## 验证

* 构造最小样例：

  * `列2 = "2025/01/19 00:58:30"，列5 = "-123.45"，列4/7/9 有文本` → 输出 `time=2025-01-19T00:58:30`，`ttype=支出`，`amount=123.45`，备注为三列合并。

* 用您当前浦发文件再次导入，导入结果弹窗显示成功条数显著提升；如仍有失败，统计为“C 列为空或无法解析”的行。

## 影响面说明

* 仅浦发的时间来源被强制固定为 C 列；其它平台不受影响。

* 若未来浦发导出模板发生变化（不再提供 C 列或格式改变），需要再做更新；当前方案确保一致性与可控性。
